About Remote Controlling laserDESK
2.1
Prerequisites
• If the Client is installed on a dedicated PC, both 
PCs must be connected with each other, see 
Chapter 2.3 ”About Installation Variants, 
Connection Variants”, page 13. 
• laserDESK must be configured as described in:
– Chapter 2.3.1 ”Configuring 
Ethernet connection”, page 15 
– Chapter 2.3.2 ”Configuring Serial Connection”, 
page 17 
• laserDESK must have been started 
• The access to laserDESK must not be blocked by 
another Remote Control User Program. 
• Only 1 Client can be connected! 
2.2
Intended Use
Remote control functionality is intended for 
adapting(1) and executing (not: designing) 
laserDESK jobs. In particular, this includes: 
• Selecting of the laserDESK job to be processed 
• Defining text content 
• Starting the execution of the laserDESK job 
For this purpose, laserDESK features a remote control 
interface. This can be addressed by: 
• Any Ethernet-capable device 
• Device with serial interface 
A master control(2) (for example, an PLC) can use this 
to remotely control laserDESK by Telegrams, see 
Figure 1: 
• see Chapter 3 ”Remote Controlling laserDESK – 
by Telegrams”, page 21 
If you have a MS Windows-based master control(2) 
you can use SLLDRemoteControl.dll instead(3). This 
simplifies programming considerably. In particular, 
there is no need to program the Ethernet interface, 
• see Chapter 4 ”Remote Controlling laserDESK – 
by SLLDRemoteControl.dll”, page 127 
Notice!
• Remote control functionality requires 
(alternativ): 
– laserDESK dongle for 
Standard + Remote Control Edition 
– laserDESK dongle for 
Premium Edition 
Refer to laserDESK Help, topics “Software 
Packages / Functional Range” and “Upgrading 
the laserDESK Software”). 
Notice!
• laserDESK jobs are executable only with an 
RTC5 board, RTC6 board or RTC6 Ethernet 
Board from SCANLAB. 
• In this manual, these boards are referred to as: 
– RTC board 
(1) Graphic objects inside of laserDESK job can only be 
changed to a limited extent but cannot be created. 
(2) In Figure 1 denoted as: 
“remote control user program”. 
(3) When using the SLLDRemoteControl.dll, Telegrams are 
generated internally.

2.4
About UIDs
laserDESK assigns an unique identifier (UID) to every 
marking object inside a job. 
Remote Commands and RemoteDLL Functions are 
able to address a certain target object exactly by 
them. 
Notes  
• The Remote Command (RemoteDLL Function) 
defines the action to be performed on the target 
object. 
• laserDESK GUI offers more options for 
manipulating the objects in the laserDESK job 
than Remote Commands and RemoteDLL 
Functions. 
2.5
About Variable Text
The string of every text object can be changed by: 
• Remote Command 14 – Set Text String 
• RemoteDLL Function SetText 
All other graphic parameters of the text object cannot 
be changed by Remote Commands or RemoteDLL 
Functions. 
The text object is selected by its UID and can be 
changed at any time. 
At job execution start, the current text string is taken 
to calculate the marking vectors. That is why Remote 
Command 14 – Set Text String and RemoteDLL 
Function SetText are rejected during this time. 
In order for the Remote Command or RemoteDLL 
Function to change the text, the graphic parameter 
'Variable' of the text object must be set to “true”. 
The texts of the following objects cannot be changed 
by Remote Command 14 – Set Text String or 
RemoteDLL Function SetText as these are updated by 
their special graphic properties: 
• Serial Number 
• Date/Time 
• Import Text

2.6
About Automatic Mode
When switching to Automatic Mode, several settings 
are made: 
• The external start directly to the RTC board is 
enabled 
• The process data of the constant (= non-variable) 
marking objects are calculated 
• The RTC board list memory is initialized and 
loaded with constant data 
• The serial numbers are reset to their start values 
(if activated) 
• The current index of a text import is reset to the

number of executions of the variants or job is 
updated. Additional the target quantity is checked

possible and laserDESK switches back to 
Manual Mode.

switching off the Automatic Mode.

• Start value 
• Actual value 
• End value

them is only possible in Manual Mode.

About Calculating Checksums
• The Checksum is calculated as the sum of all byte 
values of the data bytes. Then the cross sum of 
the digits is calculated and the last digit is taken 
as the Checksum. 
– Example Code (C#) - Short, page 22 
– Example Code (C#) - Long, page 23 
• A Checksum is always < 10. 
• In Data Part of the Telegram, you must mask even 
a Checksum = 0x02 or 0x03 by 0x10. 
• If no data has been sent, the Checksum is 0. 
Example Code (C#) - Short 
// Checksum calculation 
public static Byte GetChecksum(Byte[] arr, int len) 
{ 
Byte
cs = 0; 
int
i, h = 0, z = 0; 
// summarize every byte of array 
for (i = 0; i < len; i++)
{ 
h += arr[i]; 
} 
// calculate sum of digits 
do
{ 
z += (h % 10); 
h /= 10; 
} 
// every digit of byte 
while (h > 0);
// take last digit 
return cs = (byte)(z % 10);
}

Executing Remote Control – 
Using Remote Commands
To use laserDESK remote control with Remote 
Commands: 
(1) Start laserDESK. 
(2) Make sure that neither the pilot laser mode nor 
another program blocks the access to laserDESK. 
(3) Log in to laserDESK by Remote Command 01 – 
Login. 
The access to laserDESK is blocked for every other 
Client. 
In this state you can query: 
– laserDESK version 
– laserDESK status 
(4) Switch Remote Mode on. 
A blocking dialog opens, see Figure 6. 
Any user interaction is blocked. 
Only users with laserDESK access authorization 
“Administrator” or “Supervisor” can cancel the 
lock by the Remote Control button. 
(5) Execute the required Remote Commands.

Comment(s) 
(cont’d)
• Bit #16 is for checking whether a start has been executed. Bit #16 is set until a start 
request, see Remote Command 12 – Start Job Execution, has been received. Then 
Bit #16 is reset and stays reset until execution has been successfully finished. 
Bit #16 is reset before the answer of the start-Remote Command is sent to the Client. 
Thus, if the state query after a marking start returns Bit #16 as set, the execution is 
already finished. This is important, if the processing time is in the range of the Windows 
cycle time. Then you may never receive a Bit #16 not set.

Bit #17 is set, switching into the Remote Mode is not possible.

been aborted by one of the following conditions: 
(1) User has clicked the STOP button 
(2) External stop signal has aborted the execution 
(3) Serial control node has canceled the execution 
This signal is only valid after a job execution (RM_STATE_EXEC_DONE is set) and is reset by 
the next start or switching to Automatic Mode.

progress of Remote Command 21 – Automatic Mode On. Bit #20 is reset when the 
process has been finished. If the execution has not been successful, RM_STATE_CMD_ERR is 
set.

Comment(s) 
• Remote Command 32 – Import Vector Graphic imports a vector graphic and allows to 
define its size and position. All other parameters (marking and hatching parameters) 
are transferred from the replaced vector graphics import group or inherited from the 
job node when inserted. 
• By Remote Command 31 – Set Marking Parameter Set for Job, it is possible to 
exchange the job marking parameter and therefore, the vector graphics marking 
parameters as well. 
• If ‘width‘ and ‘height‘ are positive values, the graphics may be distorted. If one value is 
zero or negative, it is adapted proportional to the other. If both values are zero or 
negative, the file values are used with the unit [mm]. 
• If the UID is an empty string, the vector graphic file is imported as a new group getting 
the job parameters as individual (not inherited) parameter sets. Therefore, they is not 
changed by Remote Command 31 – Set Marking Parameter Set for Job later on. 
Otherwise, the file replaces the existing group, take over the group parameters 
(marking and hatching) and keep the inheritance. 
• If the UID is not empty but not defined inside the job, Remote Command 32 – Import 
Vector Graphic execution is canceled. 
• If parameter 7 is omitted, the default value is: 
– enArea is set 
– All other flags not set 
• If enScaleFactors is set, first the vector file is imported using the unit information which 
is included in the file. 
• The default unit for plt files is [0,025 mm]. 
• The default unit for all other file types is [1 mm]. 
• This unit is used for the scaled import, if no unit information is present. To import a 
plt file which has [mm] units you can either use enValues1to1 or use scaled import with 
scaling factor 40. Only the latter allows to set a different origin. 
• After the import the whole graphics is scaled by the given factors. This is different to 
enValues1to1 where no unit information is used but the file values are taken as [mm] 
values. 
• enScaleFactors allows to import a vector graphic with known measure and unknown 
size to be imported with the correct size. Negative scaling factors are ignored. 
• Symbols are always imported and updated.

36 – Change Laser Power Scaling Factor 
Purpose 
Changes the scaling factor for the laser power. 
Prerequisite 
• Only accepted, if the Client is logged on to laserDESK. 
• Only accepted, if RM_STATE_RM_MODE is set (= Remote Mode on). 
• Only accepted, if RM_STATE_EXEC_DONE is set (= no marking is executed). 
• Only accepted, if RM_STATE_LST_CALC is reset. 
• Only accepted, if RM_STATE_AUTOMODE is reset (= Automatic Mode off = Manual Mode 
on). 
Parameter(s) 
1 
[double]. 
Laser power scaling factor. 
2 
[integer]. 
1 = permanent. 
0 = temporary. 
Answer 
[integer]. 
1 = success. 
0 = error. 
Comment(s) 
• If the scaling of the laser power is to be permanent (parameter 2 = 1), the laser 
calibration factor of power in laserDESK Hardware Configuration is changed. 
This setting is also written in the hardware configuration file (active hardware 
configuration) and therefore, is used even after a program restart. 
• If the laser power is only to be scaled temporarily (parameter 2 = 0), it is reset to the 
laserDESK Hardware Configuration value when the RTC board is initialized again (on 
laserDESK start or configuration change) or when the Remote Mode is finished (to be 
consistent with laserDESK GUI). 
• The permanent setting is an absolute value. 
The temporary setting is a relative scaling factor. 
• During execution both factors are multiplied. Therefore, it is possible to define a 
temporary factor > 1.0 to increase the power level, if the permanent factor < 1.0. 
• For execution, it is checked that the total scaling factor is between 1.0…0.0.

(1) If the job has already a parameter set with given name it is replaced. 
Otherwise, it is created. 
(2) All Remote Command parameters are valid. A parameter set is created or replaced

(3) Parameter 1 (UID) is invalid. A parameter set is created or replaced in the job library 
and no assignment is changed. 
(4) Parameter 4 is omitted. An individual parameter set for the selected element is 
created. 
• Until no further error happens, any new created or replaced parameter set gets the new 
value.

– parameter ID to 0x10000002.
• Example – To select the Dithering ‘Floyd Steinberg’, set: 
– parameter ID to 0x40000002 
– parameter value to 1

Comment(s) 
• Remote Command 53 – Import and Slice stl File imports an stl File and cuts it in 
parallel slices for 2,5D processes (for example, deep engraving, additive 
manufacturing). 
Several options define the resulting laserDESK job(s). 
• If option flag 0x00000001 (Apply filling) is set: 
Filling parameters of the job node are used and the angle offset defined by parameter 
[10] is applied. You may use Remote Command 51 – Set Specific Parameter Value to 
define appropriate filling parameters prior to the slicing. 
• If 0x00000002 (Calculate slices descendant) is set: 
Slices are calculated from top to bottom. This is especially for deep engraving 
applications. 
• If 0x00000010 (Insert z-focus control nodes) is set: 
z-focus control nodes are inserted into the job instead of setting the z-value of each 
polygon point. 
• If 0x00000020 (Create layers) is set: 
Polygons of each slice are inserted in an own layer labeled ”Layer n”. 
The z-values are a layer property and not a polygon point value. 
If flag 0x00000010 is set in addition, the z-value is defined by the z-focus node 
and not by the layer. 
• If 0x00000040 (Create job for each slice) is set: 
Each slice is saved in an own job. The job name consists of the current job name with 
the extension “_Leveln.sld” where n is the number of the slice. 
• If 0x00000080 (Insert signal exchange) is set: 
All parameters are required, in particular [11] and [12]. 
• If 0x00000100 (Distance flag) is set: 
Parameter [8] is taken as the distance of the slices, not as the amount of the calculated 
slices.

ends, all transformations are cleared and the node parameters are valid again.

About SLLDRemoteControl.dll 
SLLDRemoteControl.dll enables software developers 
to easily integrate remote control of laserDESK to 
own MS Windows user programs. 
Knowledge of MS Windows sockets and TCP/IP 
connections is not required. The whole connection is 
handled by SLLDRemoteControl.dll. For executing 
tasks in laserDESK, RemoteDLL Functions just need to 
be called. 
SLLDRemoteControl.dll is written in C# with .NET and 
therefore, easily to integrate in other .NET 
user programs. 
Just add a reference into your project and use the 
SLLD_Functions class, see Chapter 4.5 ”Example Code 
(C#)”, page 205. 
Furthermore, it has been created as a COM object and 
thus can be included in C/C++ user programs as a 
COM component, see Chapter 4.6 ”Example Code 
(C++)”, page 206. 
SLLDRemoteControl.dll cannot be used to use the 
remote control interface of laserDESK with other 
systems (PLC …). In this case the standard and more 
general TCP/IP or RS232 interface must be used. Then 
all programming languages on different platforms 
with TCP/IP or RS232 support can be used for remote 
control of laserDESK. 
After laserDESK has been installed, 
SLLDRemoteControl.dll is located in: 
• C:\Program Files\SCANLAB\laserDESK\Remote

Bit #16 is set until a start request, see StartMarking, has been received. 
Then Bit #16 is reset and remains cleared until the processing has been successfully 
finished. Bit #16 is reset before the answer of the StartMarking is sent to the Client. 
Thus, if the state query after a marking start returns Bit #16 as set, the execution is 
already finished. This is important, if the processing time is in the range of the Windows 
cycle time. Then you may never receive a Bit #16 not set. 
• Bit #17 is for checking, whether the pilot laser mode is activated or deactivated. If 
Bit #17 is set, switching into the Remote Mode is not possible. 
• If Bit #18 is set, no RTC board is acquired. 
• Bit #19 shows the state of an aborted job. Bit #19 is set, if the last job execution has 
been aborted by one of the following conditions: 
(1) User has clicked the STOP button 
(2) External stop signal has aborted the execution 
(3) Serial control node has canceled the execution 
This signal is only valid after a job execution (RM_STATE_EXEC_DONE, page 154 is set) and 
is reset by the next start or switching to Automatic Mode. 
• Bit #20 is set during the process of switching on the Automatic Mode to check the 
progress of SwitchAutomaticMode. Bit #20 is reset when the process has been 
finished. If the execution has not been successful, RM_STATE_CMD_ERR, page 153 is set. 
• The status query is allowed during the execution of OpenJob/OpenJobNoWait and 
SwitchAutomaticMode to be able to check the state of these actions by 
RM_STATE_JOB_LOAD, page 153 or RM_STATE_SWITCH_AUTOMODE, page 155.

• With length > 0, width > 0, height > 0: the body is stretched to fit all dimensions. For

• options flags can be combined and define the resulting laserDESK job(s). 
– 0x00000001 Apply filling is set: 
Filling parameters of the job node are used and angleOffset value is applied. 
To define appropriate filling parameters prior to slicing ReplaceSpecificParameter 
can be used. 
– 0x00000002 Calculate slices descendant is set: 
Slices are calculated from top to bottom. This is especially useful for deep engraving 
applications. 
– 0x00000010 Insert z-focus control nodes is set: 
z-focus control nodes are inserted into the job instead of setting the z-value of each 
polygon point. 
– 0x00000020 Create layers is set: 
Polygons of each slice are inserted to an own layer labeled “Layer n“. 
z-values are a layer property and not a polygon point value. 
If 0x00000010 is set in addition, the z-value is defined by the z-focus node 
and not by the layer. 
– 0x00000040 (Create job for each slice) is set: 
Each slice is saved to an own job. The job name consists of the current job name with 
the extension “_Leveln.sld” where n is the number of the slice. 
– 0x00000080 Insert signal exchange is set: 
A digital input and output port line of the RTC board connector Extension1 is used. 
Before each slice an output signal is set to allow external handling. Then, laserDESK 
waits for an input signal to go on with the next slice. 
– 0x00000100 Distance flag is set: 
slices value is taken as the distance of the slices and not as the amount of the 
calculated slices.

Example Code (C++)
Using SLLDRemoteControl as a COM component – Important: 
SCANLAB does not provide any assistance with this programming technique! 
// ################ COM-class has to be registered (using Regasm.exe)
##########
// Regasm.exe creates the tlb file (SLLDRemoteControl.tlb) needed for the import into
// your C++ project.
// During the tlb-Import you define the class name of the C++ class and select the
// interface (here _SLLD_Functions). There may be more than one interface.
// Inside the slldremotecontrol.tlh the original C# class name of the COM interface is
// available (in case of SLLDRemoteControl.dll it is SLLD_Functions) and denotes the
// needed CLSID
HRESULT hres;
// = CoInitialize(NULL); should already be done
int
ret;
byte *ip =
new byte[4];
// loopback address
ip[0] = 127;
ip[1] = 0;
ip[2] = 0;
ip[3] = 1;
// default port of laserDESK
long port = 3000;
// Create SafeArray for OpenConnection()
SAFEARRAY** psa;
VARTYPE vt = VT_UI1;
SAFEARRAY* sa = SafeArrayCreateVector(vt, 0, 4);
psa = &sa;
BYTE *pData;
hres = SafeArrayAccessData(sa,reinterpret_cast<void**>(&pData));
memcpy(pData, ip, 4);
// Create class instance of the COM interface class
CLSID clsid = __uuidof(SLLD_Functions);
// class ID, see .tlh file
REFIID refiid = __uuidof(_SLLD_Functions);
// interface ID, see .tlh file
// pointer to the interface
_SLLD_Functions *fptr;
hres = CoCreateInstance(clsid, NULL,CLSCTX_INPROC_SERVER, refiid,(LPVOID *)&fptr);
// use the interface functions
long result = fptr->OpenConnection(psa, port);
unsigned int state = fptr->GetState();
bstr_t version = fptr->GetVersion();
WCHAR *lVersion = version.GetBSTR();
int ok = fptr->SwitchRemoteMode(true);
// add here your specific code to control laserDESK remotely
fptr->SwitchRemoteMode(false);
fptr->CloseConnection();
// clean up and free resources
SafeArrayUnaccessData(sa);
SafeArrayDestroy(sa);
sa = NULL;
}
